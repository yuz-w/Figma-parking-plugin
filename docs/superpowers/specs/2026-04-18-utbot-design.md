# utbot — Figma 사용성 테스트 플러그인 설계 문서

**작성일:** 2026-04-18  
**플러그인 이름:** utbot  
**위치:** `utbot/` (Figma-parking-plugin 레포 하위)

---

## 1. 개요

Figma에서 프레임을 선택하면 Nielsen Norman Group 10가지 휴리스틱 기반 사용성 분석 리포트를 자동 생성하는 플러그인.  
규칙 기반 정적 분석 + Claude API 시각적 판단을 결합한 하이브리드 방식으로 동작하며, 결과는 선택 프레임 오른쪽에 새 Figma 프레임으로 생성된다.

---

## 2. 파일 구조

```
utbot/
├── manifest.json
├── code.js       ← 메인 플러그인 로직 (규칙 스캔 + API 호출 + 프레임 생성)
└── ui.html       ← 패널 UI (API 키 관리 + 분석 실행)
```

---

## 3. 동작 흐름

```
1. 플러그인 실행
   └─ clientStorage에서 API 키 로드
       ├─ 없음 → API 키 입력 화면
       └─ 있음 → 분석 대기 화면

2. 사용자가 프레임 선택 + [분석 시작] 클릭

3. 규칙 기반 스캔 (code.js에서 노드 트리 순회)
   ├─ 텍스트 < 12px
   ├─ WCAG AA 색상 대비 (4.5:1 미만)
   ├─ 터치 타겟 < 44×44px (버튼/인터랙티브 컴포넌트)
   ├─ 아이콘 전용 버튼 (텍스트 레이블 없음)
   ├─ 텍스트 계층 구조 (h1/h2/body 위계 없음)
   ├─ 여백 일관성 (spacing 값 4px 그리드 벗어남)
   └─ 컴포넌트 일관성 (같은 역할의 노드가 다른 스타일)

4. 프레임 PNG 캡처
   └─ figma.exportAsync({ format: 'PNG', constraint: { type: 'SCALE', value: 1 } })

5. Claude API 호출 (claude-sonnet-4-6, vision)
   └─ 입력: base64 PNG + 규칙 체크 결과 JSON + Nielsen 휴리스틱 분석 프롬프트
   └─ 출력: JSON { insights, 기능인지, 과업흐름, 탐색유도, 오류방지, scores }

6. 리포트 프레임 생성 (선택 프레임 오른쪽 40px)
   └─ 섹션별 텍스트 노드 빌드
```

## 5. 리포트 프레임 구조

너비 600px, 높이 자동(AUTO). 배경 흰색. 섹션별로 쌓이는 VERTICAL 오토레이아웃.

| 섹션 | 내용 |
|------|------|
| 헤더 | 서비스명(프레임 이름), 분석일자, "utbot 자동 생성" |
| 규칙 체크 결과 | 항목별 ✅ 통과 / ⚠️ 경고 / ❌ 실패 배지 + 설명 |
| 핵심 인사이트 | 인사이트 / 설명 / 적용 휴리스틱 3열 테이블 |
| 분석 | 기능 인지 / 과업 흐름 / 탐색 유도 / 오류 방지 서술 |
| 종합 평가 | 항목별 1~5점 바 시각화 + Claude 총평 |

## 8. 규칙 기반 스캔 세부 기준

| 규칙 | 기준 | 심각도 |
|------|------|--------|
| 소형 텍스트 | fontSize < 12px | ⚠️ 경고 |
| 색상 대비 | WCAG AA 4.5:1 미만 (일반 텍스트) | ❌ 실패 |
| 터치 타겟 | 버튼류 width 또는 height < 44px | ⚠️ 경고 |
| 아이콘 전용 버튼 | INSTANCE 노드에 TEXT 자식 없음 | ⚠️ 경고 |
| 텍스트 계층 | 동일 레벨 텍스트 fontSize 3가지 초과 | ⚠️ 경고 |
| 여백 일관성 | x/y 좌표가 4px 그리드 미정렬 (10% 초과) | ⚠️ 경고 |
| 컴포넌트 일관성 | 같은 이름 prefix 노드의 스타일 불일치 | ⚠️ 경고 |

## 9. 기술 결정 사항

- **API 키 저장:** `figma.clientStorage.setAsync('utbot-api-key', key)` — 기기 로컬 저장, 서버 전송 없음
- **모델:** `claude-sonnet-4-6` (vision + 빠른 응답)
- **이미지 전송:** base64 PNG, max scale 1x (용량 절감)
- **색상 대비 계산:** sRGB 상대 휘도 공식으로 code.js 내에서 직접 계산 (외부 라이브러리 없음)
- **리포트 프레임 위치:** `selectedFrame.x + selectedFrame.width + 40`
